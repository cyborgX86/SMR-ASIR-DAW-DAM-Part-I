Activitats

Identificació de tipus d'adreça IP

L’objectiu d’aquesta activitat és l’identificació de classes d’adreces IP.

Identifiqueu la classe d’adreça IP de les adreces següents:

  • 68.127.23.4
  • 142.123.23.1
  • 185.23.145.233
  • 218.12.12.12
  • 92.3.23.54
  • 192.3.23.54

  • 68.127.23.4

Si teniu l’adreça IP 68.127.23.4 i voleu comprovar de quin tipus d’adreça es tracta, només cal agafar el primer octet, en aquest cas el 68 i fer la conversió a binari.

Com que s’ha dividit 68 entre dos, ha quedat un nombre de set bits, com que les IP es basen en octets cal afegir un bit amb valor “0” en el bit de major pes, per tant el nombre transformat que queda és el següent:

01000100 → 68

Com podeu comprovar, el bit de major pes d’aquest octet té un valor de “0” que és precisament el que identifica que es tracta d’una adreça de classe A, així doncs, la resta de bits (.127.23.4 en valor decimal) fan referència al nombre de host connectat a la xarxa.

  • 142.123.23.1

Si teniu l’adreça IP 142.123.23.1 i voleu saber de quina classe d’adreça es tracta, el que s’ha de fer és el mateix procediment, s’agafa el primer camp de l’adreça IP i es fa el canvi de base de decimal a binari.

El nombre obtingut és el 10001110, si mireu els dos bits de major pes comprovareu que tenen com a valor “10”, que són precisament els bits inicials de les adreces de classe B, per tant, aquesta adreça és de classe B.

  • 185.23.145.233

Si teniu l’adreça IP 185.23.145.233 i voleu saber de quina classe d’adreça es tracta, el que s’ha de fer és el mateix procediment, s’agafa el primer camp de l’adreça IP i es fa el canvi de base de decimal a binari.

El nombre obtingut és el 10111001, si mireu els dos bits de major pes comprovareu que tenen com a valor “10”, que són precisament els bits inicials de les adreces de classe B, per tant, aquesta adreça és de classe B.

  • 218.12.12.12

Si teniu l’adreça IP 218.12.12.12 i voleu saber de quina classe d’adreça es tracta, el que s’ha de fer és el mateix procediment, s’agafa el primer camp de l’adreça IP i es fa el canvi de base de decimal a binari.

El nombre obtingut és el 11011010, si mireu els tres bits de major pes comprovareu que tenen com a valor “110”, que són precisament els bits inicials de les adreces de classe C, per tant, aquesta adreça és de classe C.

  • 92.3.23.54

Si teniu l’adreça IP 92.3.23.54 i voleu saber de quina classe d’adreça es tracta, el que s’ha de fer és el mateix procediment, s’agafa el primer camp de l’adreça IP i es fa el canvi de base de decimal a binari.

El nombre obtingut és el 01011100, i com podeu comprovar el bit de major pes d’aquest octet té un valor de “0”, que és precisament el que identifica que es tracta d’una adreça de classe A.

  • 192.3.23.54

Si teniu l’adreça IP 192.3.23.54 i voleu saber de quina classe d’adreça es tracta, el que s’ha de fer és el mateix procediment, s’agafa el primer camp de l’adreça IP i es fa el canvi de base de decimal a binari.

El nombre obtingut és el 11000000, si mireu els tres bits de major pes comprovareu que tenen com a valor “110” , que són precisament els bits inicials de les adreces de classe C, per tant, aquesta adreça és de classe C.

Càlcul de binari a decimal

L’objectiu d’aquesta activitat és la familiarització en l’operació de canvi de base.

Partint de l’adreça IP en binari 11000000.10101000.00100000.01111011, calculeu quin és el nombre en decimal corresponent a aquesta adreça:

El primer pas és separar el nombre en octets, de fet, com que l’adreça IP ja està separada per ella mateixa per octets, només cal que comencem pel primer.

11000000

El que s’ha de fer és agafar cadascun dels bits i multiplicar-los per la base en la qual hi ha el nombre a transformar, elevada a la posició que ocupa dins del nombre, cal recordar que la primera posició, la de més a la dreta, sempre és 0, i s’ha de sumar les multiplicacions.

Un cop fet el primer octet, s’ha de fer la transformació del segon octet, o sigui 10101000.

Ara és el moment de fer la transformació del tercer octet 00100000.

I per completar el canvi de base només queda la transformació de l’últim octet que en aquest cas és el 01111011.

Un cop finalitzada la transformació del quart octet, només cal agafar tots els resultats i escriure’ls separats per punt, d’aquesta manera aconseguiu tenir l’adreça IP en format decimal.

192.168.32.123

Càlcul de subxarxes I

L’objectiu d’aquesta activitat és resoldre un problema concret de definició i creació de subxarxes.

Si disposeu de l’adreça privada de classe A, que té com a IP el nombre 10.X.X.X i utilitzeu la màscara de subxarxa 255.248.0.0, quina quantitat de subxarxes es poden implementar, i quants hosts s’hi poden connectar? A quina subxarxa pertany l’adreça IP 10.230.23.11?

Utilitzant aquesta màscara de subxarxa podeu comprovar que per definir les subxarxes s’utilitzen cinc bits del segon octet, això fa que es puguin implementar 30 subxarxes diferents.

Mentre que a cada subxarxa s’hi podran connectar 524.286 màquines, això vol dir que en total s’hi poden connectar 15.728.580 màquines.

A quina subxarxa pertany l’adreça IP 10.230.23.11?

Per comprovar-ho el primer que s’ha de fer és transformar l’adreça IP de format decimal a format binari.

10.230.23.11 → 00001010.11100110.00010111.00001011

Un cop es té tant l’adreça IP com la màscara en format binari apliquem l’operació lògica AND.

Un cop aplicada l’AND podeu comprovar que l’adreça de subxarxa és la següent:

00001010.11100000.00000000.00000000 → 10.224.0.0

Per tant, la resta de bits són els que estan dedicats a identificar la IP de host: 00000000.00000110.00010111.00001011 → 0.6.23.11

Càlcul de subxarxes II

L’objectiu d’aquesta activitat és resoldre un problema concret de creació de subxarxes.

Si teniu la màscara següent de subxarxa: 255.255.240.0, quantes subxarxes s’hi poden connectar? I quants hosts? Quin serà el rang d’adreces IP a utilitzar si es fa servir l’adreça privada de classe B?

Si passem aquesta màscara de subxarxa a format binari, comprovarem quants bits estan dedicats a codificar subxarxes i quants bits ho estan per host, per tant, la màscara en format binari té el valor de:

11111111.11111111.11110000.00000000

Pel que fa a la quantitat de hosts que es poden connectar a cada subxarxa, s’ha d’aplicar la mateixa fórmula, però en aquest cas el nombre de bits representen la quantitat de bits dedicats a codificar els hosts, en aquest cas hi ha la meitat del tercer octet i tot el quart, per tant, 12 bits dedicats a codificar host.

Tenint en compte que es tracta d’una subxarxa amb adreces de classe B, el rang d’adreces que es pot utilitzar per implementar aquesta subxarxa va des de 172.16.0.0 fins a 172.31.255.255 que són les adreces reservades per implementar xarxes privades amb adreces de classe B.

Creació de subxarxes III
