Activitats

Execució remota d'aplicacions gràfiques

L’objectiu d’aquesta activitat és l’execució d’aplicacions gràfiques remotes utilitzant un canal segur.

El sistema X Window permet executar un entorn gràfic de manera remota. El problema és que és un protocol no segur i que, per tant, qualsevol podria interceptar les dades, ja que s’envien com a text pla. Per solucionar-ho es pot establir un túnel mitjançant SSH que xifrarà la connexió.

Establiu una connexió segura amb un equip remot que disposi del servidor SSH i del sistema X Window i executeu una aplicació gràfica remota.

En aquest cas és important observar que en fer la connexió SSH s’ha indicat el paràmetre –X per tal d’activar el reenviament de tràfic X11. 

  1. #ssh –X raul@192.168.2.5

Un cop establerta la connexió remota, ja podem executar una aplicació gràfica, com per exemple xclock o xcalc 

  1. raul@equipremot#xclock

Accés remot mitjançant TightVNC

L’objectiu d’aquesta activitat és fer una connexió remota utilitzant un client i un servidor VNC.

Hi ha molts programaris clients i servidors que implementen el protocol VNC; de tots, un dels més utilitzats tant en distribucions GNU/Linux com en el sistema operatiu Windows és el programari TightVNC. Té llicència GPL i, per tant, és fàcilment accessible per a tota la comunitat. El que fareu serà utilitzar les eines de la distribució Linkat per instal·lar el client i servidor TightVNC (en dos equips diferents) i establir una connexió.

Recordeu que un cop instal·lat el servidor, perquè es pugui accedir des d’un equip remot s’ha d’arrencar aquest servei. En fer-ho indiqueu els paràmetres següents i escolliu els valors que considereu més adients:

  • Pantalla en la qual s’establirà la sessió per fer les connexions remotes.
  • Les dimensions de la finestra amb la qual es farà la connexió.
  • La profunditat dels colors en bits per píxel.
  • El format de la representació dels píxels.

  • Instal·lació del client (en el client):
    1. #sudo apt-get install xtightvncviewer
  • Instal·lació del servidor (en el servidor):
    1. #sudo apt-get install tightvncserver
  • Arrencar el servidor VNC amb els paràmetres adequats (això dependrà del vostre sistema) (en el servidor)
    1. #vncserver :1 –geometry 1024x768 –depth 16 –pixelformat rgb565
  • Connexió a l’equip remot indicant la IP del servidor i la pantalla on està funcionant el servidor (en el client)
    1. #vncviewer 192.168.2.3:1

Accés remot via Web mitjançant TightVNC

L’objectiu d’aquesta activitat és fer una connexió remota utilitzant un client web i un servidor VNC.

L’accés remot es pot fer també mitjançant un navegador web que suporti les miniaplicacions de Java, com el Firefox. Aquest mètode té l’avantatge que no es necessita instal·lar cap programari client, ja que els navegadors web estan habitualment instal·lats a tots els equips.

Instal·leu el component de TightVNC que permet fer connexions remotes mitjançant un navegador web i establiu una connexió remota. Recordeu que per accedir des del client, necessitareu l’adreça IP del servidor i accedir a un port determinat. No serà suficient amb el port 80, que es on s’estableixen les connexions web per defecte.

Si no està instal·lat haureu d’instal·lar el servidor TightVNC. 

  1. #sudo apt-get install tightvncserver

Arrencar el servidor VNC amb els paràmetres adequats (això dependrà del vostre sistema) (en el servidor) 

  1. #vncserver :1 –geometry 1024x768 –depth 16 –pixelformat rgb565

Instal·lar el component al servidor que permetrà accés web. 

  1. #sudo apt-get install tightvnc-java

A partir d’aquest moment ja es podrà accedir al servidor per mitjà del servidor web i del port 5801 (http://192.168.2.3:5801), i el primer que passarà és que demanarà una contrasenya.
