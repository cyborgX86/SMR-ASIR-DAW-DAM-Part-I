Control remot

L’objectiu d’aquesta activitat consisteix a conèixer i provar els protocols i els programes de control remot.

En aquesta activitat instal·larem programes de control remot per poder gestionar incidències remotament.

Cal que distingim la connexió remota per consola (pantalla només de text) i connexió remota d’escriptori (amb interfície gràfica).

Distingim entre el programa servidor i client:

  • Servidor: programa que espera una connexió remota. S’instal·la en l’ordinador objectiu, aquell que volem controlar.
  • Client: programa que es connecta a un servidor. S’instal·la en l’ordinador des del qual volem fer l’assistència tècnica.

El servidor i el client es comuniquen mitjançant un protocol. Hi ha diversos protocols.

Protocols per connexió de terminals (sense interfície gràfica)

  • TELNET: és un protocol que serveix per accedir en mode terminal, és a dir, sense gràfics a una màquina remota. El port que s’utilitza és generalment el 23. El seu principal problema és de seguretat, ja que tots els noms d’usuari i contrasenyes necessàries per entrar a les màquines viatgen per la xarxa sense xifrar. Això fa que qualsevol persona que espii el tràfic de la xarxa pugui obtenir aquestes dades i d’aquesta manera accedir també a les màquines. Proveu de connectar dues màquines utilitzant el telnet, inclús proveu-ho amb màquines que tinguin un sistema operatiu diferent.
    • Ubuntu: a les fonts d’Ubuntu teniu un servidor i un client telnet
    • Windows:
      • Servidor: KpyM Telnet/SSH Server
      • Client: PuTTY
  • TELNET sobre SSL: per evitar els problemes de seguretat una solució és fer servir telnet sobre SSL una capa d’encriptació. Proveu de connectar dues màquines utilitzant el telnet sobre SSH, inclús proveu-ho amb màquines que tinguin un sistema operatiu diferent.
    • Ubuntu: a les fonts d’Ubuntu teniu un servidor telnet sobre ssl.
  • SSH: el telnet es va deixar de fer servir quasi totalment quan, fa uns anys, va aparèixer i es va popularitzar l’SSH, que es pot descriure com una versió xifrada de telnet.
    • Ubuntu: a les fonts d’Ubuntu teniu un metapacket anomenat ssh que permet instal·lar el client i el servidor.
    • Windows:
      • Servidor: KpyM Telnet/SSH Server
      • Client: PuTTY

1. Amb l’Ubuntu

En aquesta solució suposem que tenim permisos de sudoer, sinó en teniu us podeu llegir els annexos de la unitat.

Telnet

  • Servidor:
    1. Us poseu com a usuari root: sudo su -
    2. Instal·leu el servidor telnet: apt-get install telnetd
    3. Observeu si ja està en marxa: netstat -a | grep telnet tcp 0 0 *:telnet *:* LISTEN
    4. i observeu que sí que està en marxa.
    5. El control del servidor telnet es fa des del inetd. Si volem aturar el servei cal editar l’arxiu de configuració del inetd: gedit /etc/inetd.conf
    6. comentar la línia telnet,
      #:STANDARD: These are standard services.
      #telnet stream tcp nowait root /usr/sbin/tcpd /usr/sbin/in.telnetd
    7. i actualitzar el inted amb la ordre:
      /etc/init.d/openbsd-inetd restart
    8. Per canviar el port al qual serveix cal editar l’arxiu /etc/services: gedit /etc/services
  • Client:
    1. Instal·leu el servidor telnet: apt-get install telnet
    2. Connecteu al servidor: telnet ip_servidor

Afegir una capa SSL

Com que el telnet és insegur (ja que no va encriptat) necessitem afegir-hi encriptació. Podem afegir una capa SSL o bé fer un túnel ssh.

  • Servidor:
    1. Us poseu com a usuari root: sudo su -
    2. Instal·leu el servidor telnet: apt-get install telnetd-ssl
    3. Això desinstal·larà el paquet telnetd
  • Client:
    1. Instal·leu el servidor telnet: apt-get install telnet-ssl
    2. Connecteu al servidor: telnet ip_servidor
    3. Això desinstal·larà el paquet telnet.

Telnet a través d’un túnel ssh

També podem establir un túnel ssh per sobre el qual podem executar el telnet, si en el servidor hi tenim un servidor ssh: 

  1. ssh  -P -f -L  1234:remoteserver:23 usuari@ip_servidor_remot sleep 25

Això uneix el port 1234 de la màquina client amb el port 23 del servidor. El port 23 és el del telnet.

D’aquesta manera podem executar des del client una sessió de telnet sobre ssh: 

  1. telnet localhost 1234

ssh

  • Servidor:
    1. Us poseu com a usuari root: sudo su -
    2. Instal·leu el servidor ssh: apt-get install ssh
    3. Per posar-lo en marxa: /etc/init.d/ssh start
  • Client:
    1. Connexió al servidor: ssh usuari@ip_servidor

Protocols per connexió remota d'escriptoris (amb interfície gràfica)

L’objectiu d’aquesta activitat consisteix a conèixer i provar els protocols gràfics i els programes de control remot.

Hi ha múltiples protocols per la connexió remota d’escriptoris: VNC, RDP, X11, NX… Us recomanem que proveu com a mínim tres protocols diferents.

És interessant també fer un cop d’ull a les seves prestacions a l’hora de triar-ne algun. És per això que les comparatives són de gran valor.

  • Ubuntu:
    • Combinació de vino (servidor) i vinagre (client). Fan servir VNC sobre GNOME. Tot està a les fonts. El vinagre i el vino ja venen instal·lats amb l’instal·lació estàndar.
      Protocol: VNC.
      Àmbit: Només es poden executar a GNOME però poden connectar amb altres programes VNC.
    • vnc4server xvnc4viewer són paquets de servidor i client respectivament. Tot està a les fonts.
      Protocol: VNC.
      Àmbit: Multiplataforma
    • thightvncserver xthightvnclient són paquets de servidor i client respectivament.
      Protocol: VNC.
      Àmbit: Multiplataforma
    • tsclient client terminal server per connectar a una màquina Windows. Està a les fonts.
      Protocol: RDP/VNC.
      Àmbit: Natiu Windows
    • NX (NoMachine) client, node and server. Està a NoMachine.
      Protocol: NX.
      Àmbit: Multiplataforma.
    • ssh + X11
      Protocol: ssh + X-Window
      Àmbit: Natiu Unix (amb Windows utilitzant X-Ming)
    • XDMCP
      Protocol:X-Window
      Àmbit: Natiu Unix (amb Windows utilitzant X-Ming)
  • Windows:
    • TightVNC
      Protocol: VNC
      Àmbit: Multiplataforma
    • NX (NoMachine) client, node and server. Està a NoMachine.
      Protocol:
      NX.
      Àmbit: Multiplataforma.

1. VNC

Vino & Vinagre

El més fàcil per connectar remotament Ubuntu és utilitzar els següents programes:

  • Servidor: vino
  • Client: vinagre

Aquests programes utilitzen el protocol VNC i són depenents de GNOME.

Vino & TightVNC

Si volem administrar remotament un Ubuntu des d’un Windows el que podem fer servir és:

  • Servidor Ubuntu: Vino
  • Client Windows: TightVNC 2.0.2

TightVNC & Vinagre

Si volem administrar remotament un Windows des d’un Ubuntu el que podem fer servir és:

  • Servidor Windows: TightVNC 1.3 Fixeu-vos amb els requisits. Fins que no especifiqueu una contrasenya al servidor no s’acceptaran connexions.
  • Client Ubuntu: Vinagre

TightVNC & Vinagre

Entre Windows podem fer servir el ThightVNC:

  • Servidor Windows: TightVNC 2.0.2
  • Client Windows: TightVNC 2.0.2

Hi ha molts protocols i moltes combinacions possibles. Us exhortem a que en proveu més.

En el cas que hi hagi firewalls entre mig de les connexions però disposem d’una connexió ssh. Podem fer tunnels ssh i fer passar la connexió gràfica remota pel túnel.
