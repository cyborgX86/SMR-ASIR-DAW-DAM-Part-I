Activitats

Compartició d’un directori

L’objectiu d’aquesta activitat és posar en pràctica la compartició de recursos, concretament de directoris, entre diferents màquines amb sistemes operatius lliures utilitzant l’eina Samba.

Heu de compartir un directori anomenat alumnes entre dues màquines virtuals amb Ubuntu Desktop Edition instal·lat, utilitzant Samba.

1. El primer pas serà crear el directori en una de les màquines per compartir i ubicar dins els fitxers que volem compartir. 

   sudo mkdir alumnes

2. A continuació premem el botó dret damunt del directori alumnes i seleccionem la línia Opcions de compartició | Comparteix aquesta carpeta i marquem l’opció Accés de convidat. Si no tenim el servei Samba instal·lat ens demanarà que l’instal·lem.

3. El pas següent serà accedir des de l’altra màquina virtual a la carpeta remota. Així doncs, anem al menú Llocs | Xarxa i seleccionem la icona amb el nom de la màquina virtual que conté la carpeta alumnes.

4. Al navegador de fitxers ens apareixerà la carpeta alumnes i si cliquem damunt veurem tots els fitxers continguts a dins.

Configuració del servidor Samba

L’objectiu d’aquesta activitat és que creeu grups de treball i practiqueu amb les diferents opcions de configuració d’un servidor Samba amb la finalitat de compartir diferents recursos entre màquines amb sistemes operatius lliures.

Heu de configurar el servidor Samba, mitjançant l’edició de l’arxiu de configuració o mitjançant una eina gràfica, amb els paràmetres següents:

  1. Heu de crear un grup de treball anomenat CARGOLS.
  2. El vostre servidor es veurà a la xarxa amb el nom de ESCOLA.
  3. La seguretat es controlarà a escala d’usuari.
  4. L’usuari amb una contrasenya incorrecta serà rebutjat, però si l’usuari no existeix passarà a ser un usuari convidat.
  5. Denegarem l’accés a totes les màquines de la xarxa que comencin per 10.0.
  6. Denegarem l’accés als usuaris Bernat i Biel i al grup ALUMNES.
  7. Crearem un recurs compartit anomenat CLASSE, al qual assignarem el directori /home/classe, el qual estarà disponible per a tots els usuaris i convidats, i a més d’accedir-hi hi podran també escriure.
  8. Crearem un recurs compartit anomenat CASA, al qual només podran accedir els usuaris del grup CARGOLS, però no hi podran escriure. El directori assignat serà /home/casa.
  9. Compartirem el CD-ROM amb tots els usuaris que tinguin accés al sistema.
  10. Compartirem una impressora virtual anomenada RUM amb els usuaris pertanyents al grup TORTUGUES.

El fitxer de configuració quedaria de la manera següent: 

    workgroup = CARGOLS
    netbios name = ESCOLA
    security = user
    map to guest= bad user
    host deny = 10.0
    invalid users = Bernat, Biel, @alumnes
    [CLASSE]
    path = /home/classe
    read only = no
    guest ok = yes
    browseable = yes
    available = yes
    [CASA]
    path = /home/casa
    read only = yes
    guest ok = no
    browseable = yes
    available = yes
    valid users = @CARGOLS
    [CD ROM]
    path = /media/cdrom
    guest ok = yes
    browseable = yes
    available = yes
    [RUM]
    path = /var/spool/samba
    guest ok = no
    browseable = yes
    available = yes
    valid users = @tortugues
    printable = yes

Configuració del servidor NFS

L’objectiu d’aquesta activitat és posar en pràctica la compartició de recursos, concretament de directoris, entre diferents màquines amb sistemes operatius Linux mitjançant un servidor NFS.

Heu de compartir un directori entre dues màquines virtuals amb Ubuntu Desktop Edition instal·lat mitjançant l’NFS.

L’NFS és el sistema estàndard (per defecte) que utilitza Linux (i també Unix) per compartir carpetes en una xarxa. Mitjançant NFS un servidor pot compartir les seves carpetes a la xarxa. Des dels ordinadors dels usuaris es podrà accedir a les carpetes compartides com si ens trobéssim en la mateixa màquina.

L’NFS és el sistema nadiu que utilitza Linux per compartir i accedir a carpetes compartides a la xarxa.

Des del punt de vista de la màquina virtual que actua com a servidor (ubuntu-servidor), haurem d’instal·lar l’NFS server: 

   # apt-get install nfs-kernel-server nfs-common portmap

Per instal·lar l’NFS client: 

   # apt-get install nfs-common portmap

El servei portmap és un dimoni d’assignació de ports dinàmics per a serveis NFS (entre d’altres). Té l’habilitat d’assignar un rang de ports ampli per als serveis que controla.

Un cop hàgim fet les instal·lacions, ja podem exportar directoris des del servidor NFS mitjançant la compartició d’una carpeta anomenada /REPOSITORI.

En aquest cas, la màquina client NFS (ubuntu-client) accedirà via NFS a la carpeta /REPOSITORI de la màquina que actua com a servidor.

A partir d’aquest punt és important destacar que, des del punt de vista del servidor NFS, les carpetes compartides reben el nom de directoris exportats.

Les passes per exportar un directori en el servidor són: 

   # mkdir /REPOSITORI

   # chown nobody:nogroup /REPOSITORI

En aquest cas, l’execució de la comanda chown solament s’ha d’aplicar sobre el directori compartit que acabem de crear.

A continuació, editem l’arxiu de configuració (/etc/exports) del sistema en el qual tenim els directoris exportats, això és, el fitxer de configuració del sistema en què tenim els directoris exportats. 

   # gedit /etc/exports

Un cop dins, afegirem la línia següent: 

   /REPOSITORI ubuntu-client(rw)

On ubuntu-client és la màquina que actuarà com a client NFS i amb rw estem donant permisos d’escriptura i lectura per accedir a /REPOSITORI.

Si haguéssim optat per escriure: 

   /REPOSITORI *.ciclesfp.local(ro)

Estaríem donant permís de només lectura a totes les màquines que estiguessin en el domini ciclesfp.local.

Val a dir que els permisos de compartició per NFS no exclouen els permisos del sistema Linux sinó que prevalen els permisos més restrictius. Per exemple, si una carpeta estigués compartida amb permisos NFS de lectura i escriptura però en els permisos del sistema només hi haguessin permisos de lectura, llavors, no hi podríem escriure.

Ara bé, si una carpeta estigués compartida amb permisos NFS de lectura i en el sistema es disposés de permisos de lectura i escriptura, tampoc podríem escriure-hi. Per tant, per poder-hi escriure caldria disposar de permisos de lectura i escriptura tant en els permisos del sistema com en els permisos de compartició NFS.

Per tant, si els usuaris es connectessin des d’una xarxa, l’NFS es podria configurar de manera que només accedissin a la carpeta compartida des de la xarxa en concret.

Per exemple, si només es connectessin els usuaris des de la xarxa 192.168.1.0/24, llavors per NFS permetríem l’accés només des d’aquesta xarxa.

Amb la finalitat de conèixer la informació relacionada amb els permisos d’accés, podem obtenir més informació mitjançant l’execució de la línia de comandes següent: 

   # man 5 exports

Amb la intenció que es facin efectius els canvis per cada cop que canviem l’arxiu de directoris exportats, haurem d’executar el següent: 

   # exportfs -a

També podem muntar la carpeta NFS en la màquina client (ubuntu-client). Per fer-ho, haurem de crear el directori a compartir. 

   # mkdir /DADES

   # mount ubuntu-server:/REPOSITORI /DADES

En conseqüència, ara tenim la carpeta /DADES d’ubuntu-client muntada en la carpeta /REPOSITORI que comparteix ubuntu-servidor.

Per conèixer els sistemes d’arxius que hi ha muntats (entre ells /DADES que correspon a /REPOSITORI de la màquina ubuntu-servidor) a la màquina ubuntu-client haurem d’executar la comanda següent: 

   df -h

Executant l’ordre mount també veurem tots els sistemes d’arxius muntats a la màquina ubuntu-client.

A continuació, si creem un nou arxiu a /DADES (ubuntu-client), des d’ubuntu-servidor també veurem que s’ha creat el nou arxiu, òbviament en el directori compartit associat (/REPOSITORI).

Arribats a aquest punt, si volguéssim que cada cop que s’engegués l’ordinador es muntés via NFS el directori /DADES, llavors, hauríem de definir-ho així dins del directori /etc/fstab. 

   gedit /etc/fstab

Un cop dins, afegirem la línia següent: 

   ubuntu-server:/REPOSITORI /DADES     nfs     defaults    0      0

Per verificar si el servidor NFS està funcionant correctament, cal que executem el següent: 

   rpcinfo -p

Val a dir que si surten arrencades una o més instàncies dels serveis NFS, posrtmapper i mountd, això vol dir que el servidor NFS està funcionant correctament.

En el cas del client NFS, amb la finalitat de conèixer si funciona correctament, haurem de teclejar la mateixa comanda. En aquest cas, si apareixen una o més instàncies del servei portmapper significa que el client NFS està funcionant correctament.

Per últim, cal destacar que sempre que canviem la configuració dels directoris exportats en el servidor NFS (/etc/exports), haurem de tornar a exportar els directoris i tornar a arrencar el servidor nfs: 

   # exportfs -a

   # /etc/init.d/nfs-kernel-server stop

   # /etc/init.d/nfs-kernel-server start

Taula de continguts: Ofereix una vista general de l'estructura del mòdul, que us facilitarà la navegació a través dels seus continguts.
Opcions de format: Permet configurar el format de lectura a partir de les vostres preferències, modificant la mida de la lletra, el color del fons, l'amplada de la pàgina o l'ocultació dels continguts complementaris, entre d'altres.
Imprimir: Envia el contingut seleccionat a la impressora.
Prefereits: Permet desar aquelles parts del contingut a les que us interessi accedir en un moment posterior; us permetrà anar creant un repositori personalitzat de les seccions que considereu més rellevants. Un cop l'hageu començat a fer servir se us mostrarà a sota un comptador que us informarà del nombre de preferits que s'hagin emmagatzemat fins al moment.
Comptador i llistat de preferits: Indica el nombre de preferits que s'han desat fins el moment; clicant damunt seu accedireu al llistat de preferits, i a través d'aquest podreu accedir directament als continguts que hageu emmagatzemat.
Ajuda: Aquesta opció activa la informació de les funcionalitats del web tot situant el punter del ratolí al damunt dels diferents ítems.
Mostrar/Ocultar barra lateral: Deshabilita la barra d'opcions, permetent la seva recuperació quan es desitgi.
Cerca: Introduïu les paraules clau sobre aquells conceptes que desitgeu localitzar en els continguts del mòdul. També podeu excloure un terme de la cerca tot afegint-hi el signe “-” al davant.
Capçalera: Indica l'apartat o secció que es mostra en pantalla. Tingueu present que les capçaleres de segon, tercer i quart nivell també es poden desar com a marcadors.
