Resum

Els encaminadors s’usen per poder interconnectar diferents xarxes; per tant, hi passa força trànsit. La funció principal d’aquests dispositius és la de decidir per on dirigir el trànsit rebut cap a la xarxa adequada. És una funció important, clau per al bon funcionament de la xarxa.

Un encaminador ha de proporcionar una ruta segura i fiable des de l’origen fins a la destinació per als paquets de dades que es transmeten. Treballen en la capa 3 del model OSI, fan l’encaminament basant-se en adreces IP.

L’encaminador té una estructura interna semblant a un ordinador, disposa de CPU i sistema operatiu. La memòria RAM emmagatzema l’arxiu de configuració actiu denominat running-config, les taules de resolució d’adreces ARP i les taules d’encaminament. També emmagatzema els paquets de dades mentre es decideix la millor ruta d’encaminament.

La memòria NVRAM emmagatzema l’arxiu de configuració d’inici anomenat startup-config, i encara que apagueu l’encaminador aquesta informació es manté intacta. La memòria flaix emmagatzema el sistema operatiu.

La memòria ROM conté el gestor d’inici de l’encaminador, anomenat POST, el programa Bootstrap i una versió mínima del sistema operatiu anomenada monitor de ROM o ROMMON. Les interfícies són els mòduls que connecten l’encaminador amb els diferents tipus de medis i tecnologies que hi ha. Les més habituals són les de sèrie, les de consola o les Ethernet.

El Bus de dades uneix la CPU amb les diferents interfícies. Els indicadors LED permeten conèixer l’estat de l’encaminador, poden alertar d’un possible funcionament incorrecte d’aquest.

Un encaminador té diferents interfícies. Tipus LAN: permeten connectar el nostre dispositiu a una xarxa LAN. Tipus consola/AUX: doten l’encaminador de ports en sèrie asíncrons que s’utilitzen per configurar l’encaminador. Interfícies WAN: permeten connectar a xarxes d’àrea extensa. Es distribueixen al panell posterior dels encaminadors depenent del model i del tipus.

Les característiques i prestacions dels encaminadors depenen dels diferents models i fabricants del mercat. Les més comunes són les següents. NAT: l’encaminador és capaç que tots els equips client de la nostra LAN tinguin accés a una altra xarxa amb una mateixa adreça pública. Monitoratge i depuració: els encaminadors ofereixen ordres per depurar i visualitzar diferents paràmetres de rendiment o fallades del sistema. Servidor DHCP: permet assignar adreces IP a les màquines client. Llistes d’accés: per filtrar el trànsit de la xarxa basant-se en les adreces IP, els números de port o els protocols que s’utilitzen. VPN: pot establir una xarxa privada virtual amb un altre dispositiu i xifrar les comunicacions.

En iniciar un encaminador passa per les fases: fase POST, que fa un diagnòstic d’engegada, s’executen programes de comprovació per als diferents dispositius interns de l’encaminador; fase d’arrencada, s’encarrega d’executar instruccions per tal d’inicialitzar l’IOS; càrrega de l’IOS, el sistema operatiu pot residir en diverses ubicacions, el carrega des d’aquestes; càrrega de l’arxiu de configuració, resideixen en l’NVRAM, aquests s’encarreguen de configurar totes les interfícies i les taules d’encaminament o resolució de noms.

Quan l’encaminador ha acabat la seqüència d’inici, permet modificar la configuració de diferents maneres. Es pot configurar per consola, es connecta l’encaminador des d’un PC amb un cable de consola DB9-RJ45. Aquest tipus de configuració s’utilitza quan l’encaminador no disposa d’una configuració TCP/IP. Una vegada s’ha connectat s’usa un programa d’emulació de terminal i utilitzeu els ports COM per fer les connexions a l’encaminador. Es pot configurar per connexions web, usant un navegador d’Internet: cal que escriviu a la barra de navegació URL l’adreça IP de l’encaminador. Quan el navegador es connecta amb l’encaminador cal que introduïu un usuari i la seva contrasenya per tal d’accedir a la configuració. Es pot configurar per connexions Telnet: heu de disposar de l’adreça IP de l’encaminador i que aquest tingui habilitades les terminals de Telnet. Es pot configurar mitjançant la línia auxiliar, permet accedir a l’encaminador per una connexió telefònica independentment de l’estat de la xarxa.

Els modes d’execució d’ordres en què podeu treballar són diversos.

El mode usuari permet només obtenir informació de l’encaminador, i s’identifica perquè al prompt apareix el símbol >.

El mode privilegiat permet realitzar qualsevol modificació de les diferents configuracions de l’encaminador; identificareu aquest mode pel simbol # en el prompt. Per canviar del mode usuari al mode privilegiat només s’ha d’introduir l’ordre enable. Per tornar al mode usuari des del privilegiat només cal utilitzar l’ordre disable.

Al mode de configuració global s’accedeix des del mode privilegiat amb l’ordre configure terminal. A partir d’aquest mode podeu fer qualsevol modificació en els dispositius. Identifiqueu aquest mode perquè en el prompt hi ha (config).

El mode de configuració d’interfícies us permet visualitzar i modificar les configuracions de què disposa l’encaminador. S’accedeix des del mode de configuració global amb l’ordre interface nom_interficie. Identifiqueu aquest mode perquè en el prompt apareix (config-if).

El mode configuració per consola permet configurar la connexió quan ho feu mitjançant la consola o Telnet i s’accedeix amb l’ordre line des del mode de configuració global. Sempre l’heu d’executar des del mode de configuració global.

L’ordre exit s’utilitza per passar d’un mode de configuració al mode anterior. L’ordre reload realitza una recarrega de l’encaminador. Per evitar la pèrdua de la configuració utilitzeu l’ordre copy running-config startup-config.

L’ordre show us permet visualitzar qualsevol informació de configuració del vostre dispositiu. Podeu observar diferents opcions de l’ordre executant show ?.

Mitjançant l’ordre show startup-config veieu la configuració d’inici. Aquesta s’emmagatzema a l’NVRAM per defecte. I cada cop que s’inicia l’encaminador és aquesta la configuració que es carrega.

Per donar seguretat d’accés a l’encaminador es poden definir contrasenyes per a les diverses maneres d’accés a l’encaminador. L’ordre password clau defineix la clau d’accés per entrar al mode usuari. L’ordre login activa la contrasenya abans definida. Les ordres enable password contrasenya i enable secret contrasenya permeten definir la contrasenya per al mode privilegiat però la segona la desa xifrada. Amb l’ordre line vty 0 4 accediu al mode de configuració de les línies de Telnet, i un cop aquí es poden configurar les contrasenyes.

Per assignar la configuració TCP/IP a les interfícies heu d’accedir al mode de configuració global de l’encaminador i a continuació a la configuració de cadascuna de les interfícies. L’ordre ip address ip màscara configura els valors TCP/IP en qualsevol tipus d’interfície, i s’activa amb l’ordre no shutdown.

La connexió entre encaminadors es fa mitjançant les interfícies en sèrie connectant un cable anomenat DTE-DCE. Un dels encaminadors fa les funcions de DTE (interfície en sèrie 0) o equip de terminació de dades i l’altre fa les funcions de DCE (interfície en sèrie 1) o equip de terminació del circuit de dades. Amb l’ordre clock rate velocitat es configura la velocitat de transmissió entre aquests. L’encaminador que gestiona la transferència i determina la velocitat de transmissió s’anomena DCE.

Els arxius de l’IOS i els de configuració d’inici són els elements fonamentals per al bon funcionament del vostre encaminador. A vegades us pot interessar modificar el procés d’inici de l’encaminador. Per fer-ho teniu dos mètodes: un és modificar el registre de configuració, i l’altre usar les ordres boot system. El monitor de ROM o ROMMON és una versió mínima del sistema operatiu amb un conjunt de funcions limitades i permet restaurar la contrasenya, modificar el registre de configuració, gestionar la seqüència d’inici de l’encaminador, restaurar còpies de seguretat de l’IOS amb TFTP i restaurar còpies de seguretat de l’IOS amb Xmodem.

El servei d’TFTP permet disposar de còpies de seguretat per tal de recuperar una imatge de l’IOS o d’un arxiu de configuració. Per accedir des de l’encaminador al servidor TFTP només us cal conèixer-ne l’adreça IP del servidor i fer la còpia de seguretat de l’IOS o de l’arxiu de configuració.

L’encaminament IP és una de les funcions fonamentals dels encaminadors. Consisteix fonamentalment a determinar quina és la ruta que ha de seguir un paquet de dades d’un ordinador d’origen fins a un ordinador de destinació basant-se en els factors: nombre de salts de l’origen a la destinació, amplada de banda de la línia, nombre d’usuaris connectats i prioritats.

Les rutes estàtiques són aquelles que l’administrador introdueix en els encaminadors manualment, i això comporta que s’hagi de tenir un profund coneixement de la xarxa que esteu configurant. A més, és un sistema que no varia amb el temps i no s’actualitzen les taules d’encaminament d’una manera automàtica quan hi ha modificacions en la xarxa. Per poder veure la taula d’encaminament d’un encaminador teniu l’ordre show ip route.

Les rutes estàtiques s’introdueixen en el sistema amb l’ordre ip route identificador_de_xarxa mascara porta_enllaç distancia_administrativa . L’ordre no ip route permet eliminar les entrades de rutes estàtiques que s’han generat.

Les rutes per defecte permeten substituir totes les rutes estàtiques que tenen una mateixa porta d’enllaç, i això implica menys tasca administrativa que si heu d’especificar cadascuna de les rutes. Per configurar una ruta per defecte utilitzeu l’ordre ip route amb l’identificador de xarxa a 0 i la màscara a 0.

Els encaminadors poden també funcionar com a servidors DHCP. Per fer-ho es crea un magatzem d’adreces amb l’ordre ip dhcp pool nom_magatzem. En executar aquesta ordre s’entra al mode de configuració de DHCP (Router(dhcp-config)#), un cop aquí amb l’ordre network adreça_xarxa mascara_xarxa per fixar el rang d’adreces disponibles; default-gateway per a l’adreça IP de la porta d’enllaç; lease {dies [hores] [minuts] | infinite} per a la duració de l’assignació de l’adreça; dns-server adreça_ip per a l’adreça IP del servidor DNS. Per assignar adreces fixes es fa amb l’ordre ip dhcp excluded -address adreça_inicial [adreça_final]

Les llistes de control d’accés (ACL) permeten filtrar el trànsit que travessa l’encaminador. Es pot filtrar per protocol o número de port i permetre (permit) o denegar (deny) el trànsit. Una ACL és una seqüència d’instruccions que permeten definir i filtrar el trànsit que arriba o surt d’un encaminador, permeten o deneguen l’accés a xarxes, aplicacions, protocols i ports. L’ordre access-list permet crear-les. L’ordre no access-list esborra l’ACL.

Les ACL estàndard són llistes de control d’accés que només permeten filtrar el trànsit IP. Es basen en les adreces d’origen, no tenen en compte la destinació de les dades.

Les llistes esteses són les que permeten filtrar el trànsit basant-nos en paràmetres com poden ser la xarxa d’origen o destinació, l’equip d’origen o destinació, el protocol o l’aplicació.

Les ACL anomenades són llistes de control d’accés que us permeten identificar-les amb un nom comú; això garanteix una administració molt més àgil, ja que es reconeixen ràpidament.

El diagnòstic de fallades és el procés que permet que l’administrador de la xarxa tingui les eines necessàries per determinar quins són els problemes que teniu en una xarxa. És important que disposeu d’un procés de diagnòstic i resolució de problemes estructurat i amb una seqüència determinada.

Les fallades més comunes en les xarxes es produeixen per una connexió deficient dels dispositius de xarxa amb els diferents medis de transmissió; les proves de capa física i d’enllaç determinen aquests problemes. Les fallades de la capa física són les més comunes, normalment es produeixen per cables defectuosos o incorrectament connectats. Les fallades més comunes de la capa d’enllaç es produeixen per una configuració incorrecta de l’administrador de la xarxa, o la no-activació de les interfícies. Les fallades de la capa de xarxa poden ser configuracions incorrectes de TCP/IP o perquè els protocols d’encaminament no funcionen correctament; també pot passar que les llistes de control d’accés siguin incorrectes.
